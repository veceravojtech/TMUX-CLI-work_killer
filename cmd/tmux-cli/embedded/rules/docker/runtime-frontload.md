# Docker runtime conventions (pack: docker — RUN_TARGET=docker)

Binding planning convention, loaded when the project runs in docker mode.
Extracted verbatim from the planner's `<conventions>` block.

<rule critical="true" id="DOCKER-RUNTIME-FRONTLOAD" condition="RUN_TARGET=docker">In docker mode goal-002 MUST first MATERIALIZE AND START the dev runtime container BEFORE running any PHP-toolchain command. Prepend a fan-out task "task-R" (runtime, prerequisite, runs before task-0): create docker-compose.yaml (service {{APP_SVC}} built from a dev Dockerfile carrying PHP {{php_version}} + the required extensions, plus the DB/services) and that Dockerfile, then start the stack with `docker compose up -d --build`. Publish HOST-REQUIRED ports only: the app web port from the per-project offset recorded in test-environment.md Published Ports (e.g. `18080:80`). db/mailer/redis stay compose-internal — reached intra-compose by service name (e.g. db:5432) with NO host binding — so they cannot clash on the host. Treat the HOST side of each published mapping as REBINDABLE: the service/container port stays authoritative, so a host-port clash remaps to the next free host port (e.g. `18081:80`) instead of aborting `docker compose up -d --wait`. task-0 (composer) and tasks 1-2 then add depends_on=task-R; their PHP commands are emitted BARE and the daemon wraps them into {{APP_SVC}} at dispatch (CMD-CONV). This is the DEV runtime — it replaces the late-stage compose goal for development; step 3.26 is reserved for PRODUCTION deployment extras (ADR-driven) only. Add SC-00 "docker-compose.yaml + dev Dockerfile exist and `docker compose up -d` brings {{APP_SVC}} up healthy" to the acceptance array as the first criterion. ADDITIONALLY when HAS_FRONTEND==true / Playwright is applicable: task-R MUST also add the {{NODE_SVC}} service (default "e2e", FROM an official Playwright image, same compose network as {{APP_SVC}}, with the app source mounted) so the daemon can route bare browser/E2E commands into it — and extend SC-00 to require {{NODE_SVC}} up too. When API-only, do NOT add {{NODE_SVC}}.</rule>
