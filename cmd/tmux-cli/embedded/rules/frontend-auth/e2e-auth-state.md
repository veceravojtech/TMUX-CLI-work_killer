# Authenticated E2E state reuse (pack: frontend-auth — HAS_FRONTEND AND N_auth_flows>=1)

Binding planning convention, loaded when the project has a frontend AND at
least one auth flow. Extracted verbatim from the planner's `<conventions>`
block.

<rule critical="true" id="E2E-AUTH-STATE-CONV" condition="N_auth_flows>=1 AND HAS_FRONTEND">AUTHENTICATED E2E STATE REUSE — BINDING. When auth flows exist and Playwright is available, a single Playwright `setup` project logs in ONCE per role and writes storageState JSON to a gitignored deterministic path — all other authenticated E2E tests reuse the stored state instead of logging in per test. Concretely: (1) SCAFFOLD CONFIG — goal-002's playwright.config.ts (task-2, condition: HAS_FRONTEND AND N_auth_flows>=1) MUST define a `setup` project running playwright/auth.setup.ts that writes storageState to playwright/.auth/admin.json and playwright/.auth/user.json, plus an `authenticated` project declaring `dependencies: ['setup']` and `use: { storageState: 'playwright/.auth/admin.json' }`; acceptance criterion SC-21. playwright/.auth/ MUST be gitignored. (2) AUTH SETUP DELIVERY — the auth-bootstrap goal (step 3.16a, condition: HAS_FRONTEND) MUST include playwright/auth.setup.ts in its deliverables: the script reads credentials by env-var NAME from docs/architecture/test-environment.md (admin_identifier_env, admin_password_env — never hardcoded, GM-05), logs in as admin and regular user against the APP_ENV=test stack (E2E-ENV-CONV), and writes both storageState JSON files; acceptance criterion AU-11. (3) CONSUMER — every controller-action/E2E goal (step 3.18) whose action is auth_required MUST instruct the implementer to place the E2E test in the authenticated project (inherits storageState) instead of adding inline login steps. (4) PRESERVATION — auth-flow E2E specs (step 3.19) test the REAL login/register/logout UI and MUST NOT use storageState; storageState is a precondition shortcut for OTHER suites only. (5) STATE FRESHNESS — the setup project re-runs on every `npx playwright test` invocation (Playwright project dependencies always execute); ENSURE-STACK-CONV reseeds the DB before the Playwright run, so DB-backed sessions from a prior invocation are wiped — the always-re-auth default is the safe mandate.</rule>
